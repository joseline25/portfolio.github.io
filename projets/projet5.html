Résumé

Analyse statique et dynamique d’un exécutable suspect réalisée dans une VM isolée. Objectifs : comprendre le flux d’exécution, identifier comportements indésirables, détecter techniques anti-analyse (ex. IsDebuggerPresent) et proposer des patchs pédagogiques pour neutraliser les effets indésirables. Une seconde partie documente un cas d’injection mémoire (offuscation XOR → VirtualAlloc → déchiffrement → exécution).

Objectifs détaillés

  • Comprendre le fonctionnement réel du programme depuis le binaire (désassemblage / CFG / decompilation).
  • Identifier fonctionnalités indésirables (boucle infinie, exécution de commandes, DoS local).
  • Détecter et contourner de façon contrôlée les mécanismes anti-analyse (IsDebuggerPresent, CheckRemoteDebuggerPresent).
  • Proposer et appliquer un patch pédagogique pour neutraliser la boucle infinie et l’anti-débogage.
  • Documenter le mécanisme d’injection mémoire et expliquer les contre-mesures défensives.

Démarche & Outils (non-opérationnel)

  1. Observation comportementale — exécution contrôlée, surveillance CPU/I/O, captures réseau locales.
  2. Analyse statique — désassemblage et décompilation (graphes CFG) avec IDA Pro / Ghidra.
  3. Analyse dynamique — debugging en VM isolée (x64dbg / WinDbg) pour tracer appels API sensibles (IsDebuggerPresent, VirtualAlloc, system).
  4. Patching pédagogique — exemple de pattern montrant comment neutraliser un test anti-débogage sans fournir d'offsets ni binaire.
  5. Analyse d’injection mémoire — description du flux : offuscation XOR → VirtualAlloc (+PROT_EXEC) → copie → déchiffrement → exécution → nettoyage.

Outils (usage en laboratoire uniquement)

IDA Pro / Ghidra, x64dbg / WinDbg, Python (scripts pédagogiques non-exécutables), Process Explorer, Procmon, VM isolée avec snapshots.

Recommandations & contre-mesures (opérationnelles)

  • Isoler la machine suspecte (déconnecter du réseau, snapshot de la VM).
  • Surveiller et générer des alertes sur appels système sensibles : VirtualAlloc avec droits d’exécution, CreateRemoteThread, WriteProcessMemory.
  • Déployer un EDR capable de détecter les exécutions en mémoire et patterns d’offuscation .

Compétences démontrées

  • Rétro-ingénierie (désassemblage, graphes de contrôle).
  • Analyse de malware : offuscation, injection mémoire.
  • Documentation forensic (rapports et recommandations).
